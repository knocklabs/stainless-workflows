# Finish cutover

_Work through the cutover checklist, uninstall the Stainless GitHub app, and mark the migration as complete._

Once your `stlc` workflow is running green, finalize the cutover: clean up the SaaS-side automation, uninstall the Stainless GitHub app, and mark the migration as complete.

## Cutover checklist

Come back to this checklist once your [generate workflow](./11-codegen.md) and [promote workflow](./12-promote.md#choosing-how-to-promote) are running green. These steps sever the Stainless app's remaining access to your repos and finalize the cutover.

- **Do a full round-trip dry run.** Open a pull request in your config repo that touches the spec. Confirm the optional [preview workflow](./11-codegen.md) (if you have it enabled) posts its diagnostics, and that merging the PR triggers a new `stlc build` and a promote to every production repo. Completing the promote (a fast-forward push, or merging the merge-commit PR) should publish as usual.
- **Retire any SaaS-only CI secrets.** Remove `STAINLESS_API_KEY` (or equivalent) from repos and workflows that no longer call the SaaS.
- **Uninstall the Stainless GitHub App.** See [Uninstall the Stainless GitHub app](#uninstall-the-stainless-github-app) below. After uninstalling, the app can no longer push to `next`, open release PRs, or comment on pull requests. All automation runs through your own workflows from here on.
- **Archive the Stainless app project** (optional). The project, its config history, and its bundle export remain available for reference. Archive or delete it when you are confident the `stlc`-driven workflow is stable.
- **Announce the cutover to your team.** Make sure anyone who used to edit custom code or review release PRs in the Stainless app knows that all SDK work now flows through the config repo and `stlc`.

## Uninstall the Stainless GitHub app

You may want to uninstall the Stainless GitHub app if you are migrating from the Stainless app to `stlc`, changing GitHub organizations, or no longer using Stainless. Uninstalling only revokes the app's access to your repos; the repos themselves and any code Stainless has pushed are not touched.

You uninstall the app from GitHub, not from the Stainless dashboard:

1. On GitHub, open [**Settings > Applications > Installed GitHub Apps**](https://github.com/settings/installations) for personal installations, or **Your organizations > &lt;your org&gt; > Settings > GitHub Apps** for org-wide installations.
2. Find **Stainless** in the list and click **Configure**.
3. Scroll to the bottom of the configuration page and click **Uninstall**. Confirm when prompted.

After uninstalling:

- The Stainless app can no longer push to your production repos, open release PRs, or comment on pull requests. Any scheduled or triggered workflows in the Stainless dashboard that depend on the app stop working.
- Your `production_repo` settings in `stainless.yml` remain unchanged. If you reinstall the app later and grant access to the same repos, automated publishing resumes where it left off.
- Continue publishing and integrating custom code with `stlc build`.

> **Note:**
>
> Uninstalling the app does not delete your Stainless project or its history. You can reinstall later from the [Stainless dashboard](https://app.stainless.com) the same way you installed it the first time.

## Mark migration as complete

Once everything above is done, choose what happens to your Stainless organization. Both options stop billing.

- **Disable changes and builds (read-only).** Your project, configs, and bundle export stay available for reference but cannot be edited. You can come back later if you need to re-export a bundle or revisit historical config. Reach out to support if you want to fully reactivate the org.
- **Shut down (org offline).** Permanently shut down the Stainless organization. Project history and bundle exports are removed. This is not reversible.
